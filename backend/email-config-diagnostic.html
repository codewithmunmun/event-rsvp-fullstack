.env File Fix

Resolving the "Missing credentials for 'PLAIN'" error

Issue Found

Your EMAIL_PASS value contains spaces but is not wrapped in quotes.

# Your current .env file (problematic) EMAIL_PASS=pusi pfcm xvwv ewfe # ← Spaces without quotes cause issues

Environment variables with spaces need to be wrapped in quotes to be parsed correctly.

How to Fix It

1Edit Your .env File

Open your .env file and wrap the EMAIL_PASS value in quotes:

❌ Incorrect

EMAIL_PASS=pusi pfcm xvwv ewfe

✅ Correct

EMAIL_PASS="pusi pfcm xvwv ewfe"

2Your Complete .env File

After fixing, your .env file should look like this:

DB_USER=postgres DB_HOST=localhost DB_NAME=event_manager DB_PASSWORD=postgres098 DB_PORT=5432 JWT_SECRET=mySecretKey!098 EMAIL_USER=munmunkum8210@gmail.com EMAIL_PASS="pusi pfcm xvwv ewfe" QR_SECRET=munmunqr987 UPLOAD_PATH=./uploads

3Save and Restart

Save the .env file and restart your server:

# Stop your current server (Ctrl+C) # Then restart it: npm run dev

✅ The environment variables are only loaded when the server starts, so restarting is essential!

Why This Happened

Environment Variable Parsing

Environment files parse values differently based on formatting:

  • KEY=value - Basic key-value pair
  • KEY="value with spaces" - Values with spaces need quotes
  • KEY='value with spaces' - Single quotes also work

What Was Happening

Without quotes, your app password pusi pfcm xvwv ewfe was being interpreted as:

EMAIL_PASS=pusi # Only the first word was being used # The rest (pfcm xvwv ewfe) was being ignored or causing errors

What Will Happen Now

With quotes, your entire app password will be read correctly:

EMAIL_PASS="pusi pfcm xvwv ewfe" # The entire string is now the value

Testing the Fix

Create a Test Script

To verify everything is working, create a test file:

// test-env.js require('dotenv').config(); console.log('Checking environment variables...'); console.log('EMAIL_USER:', process.env.EMAIL_USER); // Check if password is set (without revealing it) if (process.env.EMAIL_PASS) { console.log('EMAIL_PASS: *** set ***'); console.log('Length:', process.env.EMAIL_PASS.length); } else { console.log('EMAIL_PASS: NOT SET'); process.exit(1); } // If the length is 16, it's correct (without spaces) // If the length is 19, it's correct (with spaces) console.log('Expected length: 19 (16 characters + 3 spaces)');

Run the test with:

node test-env.js

✅ If the length shows 19, your app password is being read correctly!
